## Projects
### Computer Accessories request
**Technologies** Powershell, ITSM, External Vendor
- **Overview** This automation streamlines the process of requesting computer accessories for total ammount up to $1k (e.g., monitors, keyboards, mice) through the ITSM system.
  
- **Process Flow:**
- Users submit accessory requests via the ITSM portal, specifying the required items and delivery address.
- The automation validates the request and forwards the order details directly to the external vendor for fulfillment.

- **Benefits:**
- Reduces manual processing time by Helpdesk team
- Ensures accurate and timely order placement
- Enhances user experience through a simplified request workflow

### Purchase Requsition autoamtion process
**Technologies** Powershell, ITSM, SAP
- **Overview:** This automation facilitates the end-to-end handling of purchase requisition requests submitted through the ITSM system, ensuring seamless integration with SAP for purchase order generation.
- **Process Flow:**
- A new purchase requisition is initiated by the requester via the ITSM portal.
- The approval workflow is dynamically determined based on the associated cost center and the total requested amount.
- The approval matrix is continuously synchronized with the HR system to ensure up-to-date approver data.
- Once the request is approved, the relevant data is automatically transmitted to SAP, where a Purchase Order (PO) document is generated.

- **Key Benefits:**
- Eliminates manual intervention and reduces processing time
- Ensures compliance with organizational approval policies
- Maintains data accuracy through real-time synchronization
- Enhances traceability and audit readiness_

### Automated Github repository creation
**Technologies** Powershell, ITSM, Github
- **Overview:** This automation facilitates the streamlined creation of GitHub repositories within organizational accounts through ITSM ticketing.
- **Process Flow:**
- Engineers initiate a request by submitting a ticket via the ITSM system, specifying the desired repository name and details.
- Designated approvers can review the request, with options to approve, deny, or suggest an alternative repository name.
- Upon approval, the automation:
- Creates the repository in the appropriate GitHub organization
- Assigns permissions to specified GitHub Teams
- Updates custom metadata or properties as required

- **Benefits:**
- Ensures governance and naming standards are enforced
- Reduces manual effort and turnaround time
- Enhances collaboration through automated team access provisioning
